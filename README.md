## Описание проекта
**GitHub User Finder** — это настольное GUI-приложение, написанное на языке Python с использованием библиотеки `tkinter`. Приложение интегрируется с официальным REST API GitHub, позволяя находить ключевую информацию о разработчиках, сохранять их в локальный список избранного и фильтровать сохраненные профили.

### Ключевые возможности:
* Моментальный поиск профилей по логину GitHub.
* Валидация входных данных на стороне клиента (защита от пустых запросов).
* Обработка ошибок сети и некорректных ответов API (404 Not Found).
* Сохранение списка избранных пользователей в файл `favorites.json` с автоматической дозаписью и чтением.
* Удаление элементов из списка избранного.
* Живая фильтрация сохраненных пользователей по подстроке.
* Современный и адаптивный графический интерфейс (использование стилей `ttk`).

## Инструкция по запуску

### Требования
* Установленный интерпретатор **Python 3.8 или выше**.
* Предустановленная библиотека `requests` для работы со сторонними API.

### Пошаговый запуск
1. Склонируйте репозиторий или скачайте исходный код:
   ```bash
   git clone <ссылка_на_ваш_репозиторий>
   cd github-user-finder
   ```
2. Установите необходимые зависимости:
   ```bash
   pip install requests
   ```
3. Запустите приложение:
   ```bash
   python main.py
   ```

---

## Использование GitHub REST API
Приложение обращается по протоколу HTTP к публичному эндпоинту:
`GET github.com{username}`

Для работы приложения не требуется передача приватных токенов авторизации (Personal Access Token), однако следует учитывать базовые ограничения GitHub на количество неавторизованных запросов (до 60 запросов в час с одного IP-адреса).

## Тестирование и примеры использования

### 1. Позитивный случай (Поиск реального профиля)
* **Действие:** Введите в поле поиска логин `octocat` и нажмите кнопку **Найти**.
* **Результат:** Текстовое поле заполнится структурированной информацией (Имя, ID, Количество репозиториев). Станет активна кнопка "Добавить в избранное".

### 2. Негативный случай (Пользователь не существует)
* **Действие:** Введите в поле поиска заведомо несуществующий логин (например, `not-found-user-xyz-999`) и нажмите **Найти**.
* **Результат:** Программа обработает ошибку `404`, выведет в интерфейс надпись "Ошибка: Пользователь не найден", кнопка добавления в избранное заблокируется во избежание сохранения некорректных данных.

### 3. Граничный случай (Валидация пустого ввода)
* **Действие:** Очистите поле ввода (или введите несколько пробелов) и нажмите кнопку **Найти**.
* **Результат:** Сработает встроенный валидатор, запрос к API заблокируется, а на экране появится модальное окно с предупреждением: *"Поле поиска не должно быть пустым."*
